Sistema de Peajes - Arquitectura de Flujos

Documento técnico que describe el flujo operativo del procesamiento de pasos y ajustes.

1. Flujo General del Paso Vehicular

Este flujo describe el proceso estándar cuando un vehículo realiza un paso por una concesión. Incluye validación, generación de transacción, cálculo tarifario y respuesta a la concesión.

flowchart TD Start([Inicio]) --> Detect[Concesion detecta paso del vehiculo] Detect --> Send[Envia informacion del paso al API] Send --> Receive[Sistema recibe payload] Receive --> Validate[Valida estructura y datos obligatorios] Validate -->|Invalido| Reject[Marca rechazo] Reject --> Log[Genera registro de intento] Log --> End([Fin]) Validate -->|Valido| CreateTx[Genera transaccion] CreateTx --> Calculate[Calcula tarifa] Calculate --> Debit[Registra debito] Debit --> Pending[Identifica pasos pendientes] Pending --> Token[Genera token confirmacion] Token --> Response[Responde a concesion] Response --> End

2. Flujo de Ajuste de Paso

Este flujo aplica cuando una concesión reporta una inconsistencia en un paso previamente registrado. El sistema valida reglas de negocio antes de generar una nueva transacción compensatoria.

flowchart TD Start2([Inicio]) --> DetectIssue[Concesion detecta inconsistencia] DetectIssue --> RequestAdjust[Envia solicitud ajuste API] RequestAdjust --> IncludeData[Incluye ID motivo y nueva categoria] IncludeData --> Receive2[Sistema recibe solicitud] Receive2 --> ValidateOriginal[Valida existencia del paso original] ValidateOriginal -->|No existe| Reject2[Rechaza ajuste] Reject2 --> NotifyError[Responde error] NotifyError --> End2([Fin]) ValidateOriginal -->|Existe| LoadJob[Carga job validacion ajuste] LoadJob --> ValidateRules[Valida reglas de negocio] ValidateRules -->|No permitido| Reject3[Rechaza ajuste] Reject3 --> NotifyConcession[Notifica concesion] NotifyConcession --> End2 ValidateRules -->|Permitido| CreateNewTx[Genera nueva transaccion] CreateNewTx --> CalcDiff[Calcula diferencia tarifaria] CalcDiff --> ApplyAdjust[Aplica debito o abono] ApplyAdjust --> History[Registra historico] History --> Confirm[Responde confirmacion] Confirm --> End2
